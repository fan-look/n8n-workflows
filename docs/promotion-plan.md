# 推广计划文档（v1.1）

更新时间：2025-11-11  
适用环境：Windows 11 用户友好、原生前端 + Node.js/Express

## 1. 目标与定位

- 目标：提升项目曝光与试用转化，吸引自动化工程师、小团队与企业内部流程管理用户。
- 定位：多语言、易部署（Windows 友好）、原生技术栈、可扩展工作流管理（本版数据库保持现状）。

## 2. 核心卖点

- 多语言（中英）完整支持，语言资源可按需加载。
- 一键部署与 Windows 兼容脚本（`npm run deploy:prod` / `scripts\deploy.bat`）。
- 数据库保持现状（本版不涉及迁移），部署与使用体验友好。
- 原生前端 + Node.js 架构，学习成本低、扩展灵活。

## 3. 受众画像与信息框架

- 受众：
  - 自动化工程师与运维人员
  - 中小团队的业务运营与技术负责人
  - 企业内 IT/数字化团队
- 信息框架（FAB）：
  - Feature：多语言、工作流管理、快速搜索、模板库
  - Advantage：Windows 友好部署、MySQL 支持、原生栈易维护
  - Benefit：降低搭建门槛、提升流程效率、便于团队协作

## 4. 推广资产（内容清单）

- 文档与指南：
  - README / README_ZH（含快速开始与部署指南）
  - docs/iteration-requirements.md（迭代需求说明）
  - UI 标准与组件使用文档（后续新增：`docs/ui-standard.md`）
- 演示与教程：
  - 演示视频（B站/YouTube）：15 分钟从安装到生产部署；工作流创建与管理演示
  - 教程文章（掘金/知乎/CSDN）：架构解析、部署与使用指南、组件与 UI 标准落地
- 示例与模板：
  - 示例工作流包（常见邮件/消息/表单集成）
  - Demo 数据与脚本（便于快速体验）

## 5. 渠道与节奏（首月）

- 第1周：
  - GitHub 优化（Topics、徽章、截图、中文 README 链接）
  - n8n 社区发帖（中/英文）与项目介绍
- 第2周：
  - 掘金/知乎/CSDN 发布教程与技术解析（系列 2–3 篇）
  - B站发布演示视频，配合文档链接与示例包下载
- 第3周：
  - 案例文章（典型业务流程改造），邀请试用与反馈问卷
  - 微信/Telegram 技术群分享与答疑
- 第4周：
  - 数据复盘与二次传播（亮点总结、FAQ 集锦）
  - 发布下一版迭代路线图（Roadmap）与招募贡献者

## 6. 内容日历与任务分解

- 周任务（示例）：
  - 周一：渠道内容准备（文稿、截图、视频脚本）
  - 周三：首发渠道发布（GitHub/n8n 社区/技术平台）
  - 周五：数据收集与反馈整理，更新 FAQ 与文档

## 7. 指标体系（KRs）

- 曝光：
  - GitHub Stars、Issues 数量（周同比）
  - docs 访问量与停留时间
- 转化：
  - Demo 下载/运行次数
  - 部署成功率（Windows 11 环境）
- 参与：
  - 社区评论与问答数量
  - Pull Request 数量与质量

## 8. 数据收集与追踪

- GitHub 与文档站点（静态页面）结合 UTM 链接与统计脚本（不涉及隐私过度收集）。
- 每周输出看板截图与结论（问题清单、优化项、下周计划）。

## 9. 风险与预案

- 文档与代码不一致：设专项检查与修复（部署脚本一致性）。
- Windows 环境差异：准备常见问题与解决方案（批处理/PowerShell 指南）。
- 内容产能不足：建立模板化写作与截图/视频录制流程，分工明确。
